# `usage.json` reference

The per-sandbox proxy sidecar writes `usage.json` to the run's output directory after every upstream API request, using an atomic write-then-rename pattern (see `internal/proxies/proxycommon/usage.go`). For the Anthropic provider the source-of-truth struct is `Snapshot` and `ModelReport` in `internal/proxies/anthropic/usage.go`; for the OpenAI/codex provider it is the identically shaped `Snapshot` and `ModelReport` in `internal/proxies/openai/usage.go`. The file is overwritten after each request so it always reflects the cumulative totals for the run so far. After the agent sandbox exits, demesne copies the file from the sidecar-private results directory to `<output_dir>/usage.json` so it appears alongside the agent's artefacts. For the broader context on how cost figures are produced and what they mean, see [Indicative cost reporting](../explanation/key-concepts.md#indicative-cost-reporting) and [Trust boundary, agents, and the per-sandbox sidecar](../explanation/trust-boundary.md).

## Fields

The top-level object and each per-model entry share the following structure.

### Top-level (`Snapshot`)

| Field | JSON tag | Type | Units | Description |
|-------|----------|------|-------|-------------|
| Total cost | `cost_usd` | number | USD | Sum of `cost_usd` across all models seen in this run. Indicative â€” see Notes. |
| Per-model breakdown | `per_model` | object | â€” | Map from model ID string to a `ModelReport` object. Keys are the Anthropic or OpenAI model identifiers as returned by the API (e.g. `"claude-sonnet-5"`, `"gpt-5.5"`). |

### Per-model entry â€” Anthropic provider (`ModelReport`)

| Field | JSON tag | Type | Units | Description |
|-------|----------|------|-------|-------------|
| Input tokens | `input_tokens` | integer | tokens | Standard (non-cached) prompt tokens for this model across the run. |
| Output tokens | `output_tokens` | integer | tokens | Completion tokens generated by this model across the run. |
| Cache-creation input tokens | `cache_creation_input_tokens` | integer | tokens | Prompt tokens written into the prompt cache this run. Billed at ~1.25Ă— the standard input rate. |
| Cache-read input tokens | `cache_read_input_tokens` | integer | tokens | Prompt tokens served from the prompt cache this run. Billed at ~0.1Ă— the standard input rate. |
| Model cost | `cost_usd` | number | USD | Indicative cost for this model entry computed from demesne's embedded pricing table (see `internal/proxies/anthropic/pricing.go`). |

### Per-model entry â€” OpenAI/codex provider (`ModelReport`)

The OpenAI `ModelReport` differs from the Anthropic one: it omits the two cache-write/read fields and adds `cached_tokens` and `reasoning_tokens` instead, matching the OpenAI Responses API usage shape.

| Field | JSON tag | Type | Units | Description |
|-------|----------|------|-------|-------------|
| Input tokens | `input_tokens` | integer | tokens | Total prompt tokens (including cached) for this model across the run. |
| Output tokens | `output_tokens` | integer | tokens | Completion tokens generated by this model. |
| Cached tokens | `cached_tokens` | integer | tokens | Subset of `input_tokens` served from the prompt cache; billed at a reduced rate. |
| Reasoning tokens | `reasoning_tokens` | integer | tokens | Subset of `output_tokens` consumed by chain-of-thought reasoning steps; billed as standard output. |
| Model cost | `cost_usd` | number | USD | Indicative cost for this model entry (see `internal/proxies/openai/pricing.go`). |

## Notes

- **`cost_usd` is indicative.** Demesne computes it from an embedded per-model pricing table and the token counts reported by the upstream API. For Claude Code OAuth, the user is billed against a Claude Console subscription rather than on a per-request basis, so the figure is useful for relative cost tracking but is not what is actually charged. The same caveat applies to the OpenAI/codex provider, where ChatGPT-OAuth is subscription-based.
- The Anthropic `usage.json` is written to `<output_dir>/usage.json` â€” the same directory returned as `output_dir` in the tool result. The sidecar writes it first to a private `sidecar-results/` directory and copies it to `/out` after the agent exits.
- The OpenAI/codex case (when a codex model is used) emits the same top-level shape (`cost_usd` + `per_model`), but each `ModelReport` entry carries `cached_tokens` and `reasoning_tokens` instead of the Anthropic cache fields.
- If the sidecar never observed a usage block (e.g., the agent exited before any API call completed), `usage.json` may not exist or may reflect only the requests that did complete.

## Example

Anthropic provider, one `claude-sonnet-5` run with prompt-cache activity:

```json
{
  "cost_usd": 0.0124,
  "per_model": {
    "claude-sonnet-5": {
      "input_tokens": 1820,
      "output_tokens": 312,
      "cache_creation_input_tokens": 4096,
      "cache_read_input_tokens": 12288,
      "cost_usd": 0.0124
    }
  }
}
```
