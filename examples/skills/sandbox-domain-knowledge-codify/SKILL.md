name: sandbox-domain-knowledge-codify
description: Convert founder/team domain expertise into a proprietary knowledge substrate — mine a mounted expertise corpus (call notes, support threads, postmortems, docs, elicitation answers) for knowledge atoms (jargon, regulatory gotchas, edge cases, frustrations, "why the obvious answer doesn't work here"), curate them into a structured searchable context library with a CLAUDE.md-style index and per-entry retrieval metadata, then codify the recurring workflows found in the corpus ("how I audit a commercial lease"-style routines) as new SKILL.md files, each proven deterministic by two blind replay children running it against the same fixture and a fresh judge checking their outputs agree. Emits an elicitation questionnaire when the corpus is thin. Apply when the founder says "capture what's in my head", "turn my expertise into context files the AI can use", "codify how I do X into a repeatable routine", "build our proprietary knowledge base". Skip when the ask is turning observed edge cases into scenario tests (sandbox-moat-test-suite), triaging founder operational load into automate/delegate buckets (sandbox-founder-load-audit), writing the build-time architecture charter (sandbox-mvp-architecture-charter), or answering a one-off analytical question over a corpus (sandbox-corpus-map-reduce).

Turn the expertise living in the founder's head and files into two durable artefacts: a topic-organized context library any future agent session can load, and codified recurring-workflow skills that run the same way every time. You author one orchestrator prompt, launch a single slow-tier `sandbox_agent`, and it runs mine → curate → codify → replay-verify autonomously. The deliverable is `/out/library/` + `/out/skills/` + `/out/REPORT.md`; there is no code landing. Knowledge comes from the founder's corpus, never the open web — a fact a generic competitor can google is not moat.

**Watch out (cross-cutting):** The killer failure is a codified routine that reads well but is under-specified — it only surfaces when two agents run it and diverge, so the blind double-replay is never skipped, and a routine that still diverges after the fix cap ships marked DRAFT, never silently as PASS. Second: the librarian must not invent domain facts — every library entry is either evidence-backed (verbatim quote + source path) or explicitly marked `asserted`. Third: the orchestrator copies deliverables into its own `/out` itself; a `sandbox_script` child dispatched to do it writes only to its own `/out/child/<name>` and strands everything.

## Procedure

1. **Intake and mode select** (orchestrator's own process). List `/in/<expertise>` and write `/workspace/manifest.jsonl` (`{"path":…,"size_bytes":…,"type":…}` per item). If no corpus is mounted or it is trivially thin, do not fabricate: write `/out/ELICITATION.md` (the questionnaire below) and a thin-corpus `/out/REPORT.md` telling the founder to answer it and re-run with the answers mounted, print `DONE`, stop. Answered questionnaires are corpus items like any other. If a previous run's library is mounted (e.g. `/in/library`), note it — step 4 merges into it rather than rebuilding.

2. **Schema and shards** (orchestrator's own process). Write `/workspace/op.md`: the locked knowledge-atom JSONL schema (prompt item 2) verbatim, plus mining instructions. Shard the manifest by context budget into `/workspace/shard-NN.jsonl` — short notes/threads ~20–50 per shard, long docs/postmortems ~10. If the corpus contains PDFs, `.docx`, or `.eml`, first spawn one `sandbox_script` parser (`name=parse`, `image=python`, `egress=package-managers`; `pdfplumber`, `python-docx`, `mail-parser`) writing plain text to `/workspace/parsed/` and parse failures to `/workspace/quarantine-parse.jsonl` — logged, never silently skipped; mine children then read `/workspace/parsed/`.

3. **MINE** — one medium-tier `sandbox_agent` per shard (`name=mine-00`, `mine-01`, …; DNS-1123: lowercase letters, digits, interior hyphens, ≤40 chars). Dispatch each with `background: true`, collect `job_id`s, poll `sandbox_wait` (`timeout_seconds: 120`), keep **≤8 in flight** with a rolling window — blocking calls are issued one per turn and run strictly sequentially, so background dispatch is the only way shards run concurrently. Each child reads `/workspace/op.md` plus its shard's files and writes `/workspace/mined/mine-NN.jsonl` (schema-locked atoms; an atom without an evidence quote is dropped, not guessed) and `/workspace/mined/mine-NN-log.md` (files skipped or unparseable, with reasons). `workflow-trace` atoms additionally capture the observed step sequence, inputs, outputs, and the judgment calls the founder made.

4. **LIBRARY** — barrier: only after every mine job is terminal, spawn one slow-tier `sandbox_agent` (`name=librarian`). It merges and dedupes all mined atoms (merging into the mounted previous library if present — update entries, don't rebuild), and writes the library to `/workspace/library/`: one entry per file at `<topic>/<slug>.md` with YAML frontmatter (prompt item 3), status `evidence-backed` or `asserted`, and `INDEX.md` — CLAUDE.md-style, one line per entry grouped by topic, index only, never content. It also clusters `workflow-trace` atoms into named workflow candidates at `/workspace/workflow-candidates.md` and appends targeted gap questions (thin topics, contradictions between sources, workflows with missing steps) to `/workspace/gaps.md`.

5. **ROUTINE SELECT** (orchestrator's own process). Apply the codification test (prompt item 4) to each workflow candidate; rank qualifiers by frequency × founder-time; pick **≤5 per run**. For each, choose a real sample input from the corpus as the replay fixture and record slug (≤20 chars, so derived child names stay ≤40), sources, and fixture path in `/workspace/routines.md`. Candidates that fail the test are listed in the report with the missing ingredient — that's capture guidance, not failure.

6. **CODIFY** — one slow-tier `sandbox_agent` per routine (`name=codify-lease-audit`, …), background-dispatched, ≤8 in flight. Each reads the library, its workflow traces, and the embedded skill conventions (prompt item 5), and writes `/workspace/skills/<slug>/SKILL.md` plus `fixtures/` (the sample input, plus an expected-output-shape note). Determinism rule: no step may say "use judgment" — every decision is an explicit rule citing a library entry by relative path, and anything that can't be ruled goes in the routine's `## Escalate to a human` section.

7. **REPLAY VERIFY** — per routine, as its codify job completes: dispatch two medium-tier replay children (`name=replay-lease-audit-a` / `-b`), background, each briefed ONLY with the routine directory, the library, and the fixture — blind to each other, to the traces, and to the codifier's reasoning. Each executes the SKILL.md literally and writes its result to `/workspace/replay/<slug>/a/` (resp. `b/`). Barrier on both, then spawn a fresh medium-tier `name=judge-lease-audit` (never the codifier scoring its own work) that compares the two outputs against the routine's own output contract: substantive agreement → `PASS`; divergence → `FAIL` naming the ambiguous step. On `FAIL`, one fresh `fix-<slug>` child tightens that step and one replay round re-runs. **Cap: 2 rounds total**; still divergent → the routine ships with `status: DRAFT` and the divergence documented in `replay-report.md`.

8. **DELIVER** (orchestrator's own process). `cp -a /workspace/library /out/library`, `cp -a /workspace/skills /out/skills`, write each routine's `replay-report.md` into its skill dir, write `/out/ELICITATION.md` (base questionnaire + the gap questions from step 4 — always emitted; the substrate grows by re-running with new answers and the current `/out/library` mounted), and `/out/REPORT.md`. `/workspace` is torn down on exit; only `/out` persists. Print `DONE`.

## Writing the orchestrator prompt

Brief it as a complete document:

1. **The domain and corpus** — the vertical, what the business does, what is mounted where (`/in/<expertise>`), formats and quirks, plus any workflows the founder already names as recurring (these seed step 5).
2. **The locked atom schema** — every mined record: `{"id":…, "type": "jargon|regulatory-gotcha|edge-case|frustration|counterintuitive|workflow-trace", "topic":…, "title":…, "claim":…, "evidence": {"quote":…, "source_path":…} | "asserted", "confidence": 0.0–1.0, "keywords": […]}`. The types are the playbook's capture targets verbatim — jargon, regulatory gotchas, edge cases, frustrations, "why the obvious answer doesn't work here" — plus `workflow-trace` feeding codification. Lock it before any miner spawns; drifting shards can't be merged.
3. **Library entry format** — one entry per file, YAML frontmatter (`id`, `topic`, `type`, `keywords`, `aliases`, `sources`, `status: evidence-backed|asserted`, `captured: <date>`), body ≤300 words stating the knowledge and the costly mistake it prevents, cross-links to related entries. `INDEX.md`: `- [title](topic/slug.md) — one-line hook`, grouped by topic.
4. **The codification test** — a workflow qualifies only if: it recurs (≥3 corpus occurrences, or the founder names it recurring); its inputs and outputs are definable as concrete files; every judgment call inside it can be written as a decision rule citing a library entry; and it has a bounded escalate-to-human condition. Cap 5 routines per run.
5. **The embedded skill conventions** (quote verbatim into every codify child's prompt — the emitted skills must be self-contained): *YAML frontmatter: `name` matching the directory (lowercase, hyphens); `description` as one paragraph — what the routine does mechanically, then "Apply when …" with concrete trigger phrases, then "Skip when …"; `allowed-tools` minimal for the routine. Body: opening paragraph naming input → deliverable; `## Procedure` as numbered steps where every step names its input path, action, and output path; decision rules as explicit if/then rules citing library entries by relative path (`../../library/<topic>/<slug>.md`) — never restate library content inline, reference it; `## Output contract` as a literal file tree; `## Escalate to a human` listing the stop conditions and what to hand off. No step may say "use judgment".*
6. **The replay protocol** — two blind replays per routine, fresh judge, `PASS`/`FAIL` against the routine's own output contract, fix cap 2 rounds, `DRAFT` on residual divergence. State that replays get only the routine dir + library + fixture.
7. **The elicitation questionnaire** (written to `/out/ELICITATION.md` verbatim, plus step-4 gap questions): (a) jargon and terms of art a generalist misreads — your definition and the costly misreading; (b) regulatory/compliance gotchas — the rule, the trap, what taught you it; (c) edge cases where the standard treatment fails and what you do instead; (d) frustrations — what tools and generalists reliably get wrong in this vertical; (e) per common task, the obvious answer and why it doesn't work here; (f) for each workflow you personally run repeatedly: name, trigger, inputs, steps in order, how you actually make each judgment call, output, failure modes, frequency.
8. **The output contract** below, the fan-out mechanics (background dispatch, ≤8 in flight, `sandbox_wait`), and the step-4/step-7 barriers.

Terse prompts produce a wiki of platitudes. Over-specify the schema, the codification test, and the conventions; the miners find what the schema names.

## Output contract

```
/out/
  REPORT.md                       # see section order below
  ELICITATION.md                  # base questionnaire + targeted gap questions (always emitted)
  library/
    INDEX.md                      # CLAUDE.md-style index: one line per entry, topic-grouped
    <topic>/<slug>.md             # one knowledge entry per file, YAML frontmatter
  skills/
    <routine-slug>/
      SKILL.md                    # the codified routine, per embedded conventions
      fixtures/                   # sample input used for replay
      replay-report.md            # PASS or DRAFT, rounds used, divergence notes
```

`REPORT.md` sections in order: **TL;DR**, **Corpus coverage** (items mined / quarantined, per mine-log), **Library map** (entries per topic, evidence-backed vs asserted counts), **Routines** (per routine: replay verdict, rounds; plus candidates that failed the codification test and what's missing), **Gaps and next capture priorities**, **Accretion protocol** (re-run with new corpus + current library mounted; answer ELICITATION.md to fill gaps).

## Launching the orchestrator

- **`directories: ["<abs path to expertise corpus>"]` is mandatory for a mining run** — without it the pipeline degrades to elicitation-only and emits just the questionnaire (which is the one legitimate no-mount mode; say which you intend). Add the previous run's `library/` as a second mount on re-runs so the substrate accretes instead of resetting.
- Tier: **slow** for the orchestrator, librarian, and codify children; **medium** for miners, replays, and judges; parse is a `sandbox_script` step — no tier.
- All fan-out stages (MINE, CODIFY, replay pairs) are background-dispatched with `sandbox_wait`, ≤8 in flight; blocking children issued one per turn run sequentially. Barriers: librarian after all miners; each judge after both of its replays.
- Child names are DNS-1123 labels ≤40 chars: `parse`, `mine-00`, `librarian`, `codify-lease-audit`, `replay-lease-audit-a`, `judge-lease-audit`, `fix-lease-audit` — never `Codify_LeaseAudit`. Keep routine slugs ≤20 chars so derived names fit.
